# qr_decoder test fixtures

Reference QR targets for the Phase 1 decode-resolution / throughput sweep.
Display these on a screen and scan them with `qr_decoder`; read `id%`/`ok%` and
decode fps from the on-device HUD and serial. Generated by `btc-datagen`
(Sparrow-fidelity). See `../../../docs/qr-scanning-performance-requirements.md`.

| File | Modules | Version | Type | Role |
|---|---|---|---|---|
| `sparrow_2of3_wsh_mainnet.png` | 77 | v15 | static descriptor | below baseline (default mode) |
| `psbt_2of3_p2wsh_100in_normal.gif` | **85** | **v17** | animated UR, 98 frames @ 5 fps | **must-scan baseline** |
| `sparrow_3of5_wsh_mainnet.png` | 93 | v19 | static descriptor | high-resolution mode |

`psbt_2of3_p2wsh_100in_normal_parts.txt` holds the 98 `ur:crypto-psbt/N-98/…`
frame strings (the GIF rendered one per 200 ms = 5 fps, Sparrow's default).

The static 77 and 93 bracket the 85-module baseline, so its resolution floor is
interpolatable from them; the GIF tests the baseline directly at throughput.
BBQr (89–125) fixtures are not yet generated — needed for the high-resolution
corner (and `K_QUIRC_MAX_VERSION ≥ 27` for BBQr-max / v27).
